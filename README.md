# 公务员考试数据分析

基于 Python 生态对国考与省考职位数据（2019-2026）的全流程数据分析项目。

## 叙事逻辑

### 1. 使用什么工具实现了什么

本项目使用 **uv** 作为 Python 包管理与环境隔离工具，结合 **pandas**（数据处理）、**matplotlib + seaborn**（可视化），对 **831,737 条**公务员考试职位数据进行多维度分析。

- **数据来源**：1 份国考数据（121,893 条）+ 31 个省份省考数据（709,844 条）
- **分析维度**：年度招录趋势、省份岗位分布、学历要求、部门系统分布、竞争比等
- **产出**：5 张分析图表 + 完整统计报告

### 2. 中间解决了什么问题

- **数据整合**：国考和省考数据结构不一，通过 `exam_type` 字段统一标识，使用 pandas concat 合并全部数据
- **编码与字体**：数据含大量中文职位名、专业名，Windows 环境下配置 SimHei 字体解决 matplotlib 中文乱码
- **缺失值处理**：分析前先统计各列缺失率，竞争比字段缺失率达 88%，分析时针对性过滤
- **学历归类**：原始学历要求字段（如"仅限本科"、"本科及以上"、"本科或硕士研究生"）格式不统一，通过关键词映射统归为 8 个标准类别
- **大文件警告**：国考 CSV 达 60MB，GitHub 建议使用 Git LFS，目前以原始方式推送

### 3. 有什么样的结果

| 结论 | 数据 |
|------|------|
| 数据总量 | 831,737 条（2019-2026） |
| 本科及以上岗位占比 | **80.05%**，本科是公考绝对主流 |
| 竞争比中位数 | **15:1**，平均竞争比 40.5:1 |
| 最激烈部门 | 生态环境部（374:1）、粮食和物资储备局（330:1） |
| 招录最多省份 | 广东 **75,071 人**，河北 **67,030 人** |
| 招录最多部门 | 公安系统 **180,604 人**，国税总局 **122,849 人** |

## 运行方式

```bash
# 安装依赖
uv sync

# 运行分析
uv run python analysis.py

# 启动 Jupyter Notebook（交互式探索）
uv run jupyter notebook
```

分析结果图表保存在 `output/` 目录下。

## 技术栈

- **Python 3.10+**
- **uv** — 包管理与虚拟环境
- **pandas** — 数据清洗与统计分析
- **matplotlib + seaborn** — 数据可视化
